# Phase 2 目標（可直接放入 /goal）

## 一、完成條件

- 本機回歸：`team status/say/inbox` + `team/watch/steer/abort` + MCP team/control tools + `codex/claude/opencode/agy` 受控啟動 + `supervise` + `install/uninstall` + `acceptance-single-machine.ps1` + `smoke.ps1` 全部通過。
- 跨機可控：`watch/steer/abort/team` 支援 `--node` 與 `member@node`；`--node local` 強制本機；無效節點或權杖錯誤必須明確失敗（不能靜默降級）。
- 治理不變：跨機 `run` 仍需 `test pass + 至少 2 個不同 vendor 審核（min_approvals=2）` 才可 land；衝突需 `escalated` 不可自動吞掉。
- 服務化可重建：`uninstall -> install -> up/serve -> mesh -> nodes -> smoke` 可反覆成功。
- 部署範圍：主專案 5-node（m1 conductor）+ 4 顆衛星（`codex` + `claude`）可重跑，且不需手工複製檔案/手工路由。

## 二、驗收切片與測試

### Slice A：控制面
- `ensemble team status --team main --json`
- `ensemble nodes`
- `ensemble watch codex@m1 --team main --since 0 --json`
- 驗收：遠端路由可用；錯誤路徑回傳非零錯誤碼且訊息清楚。

### Slice B：跨機 run
- `ensemble run "<task>" --crew crew-main.toml --repo <repo> --team main --watch main`
- `ensemble watch main --follow`
- `ensemble steer main "<prompt>" --team main` 或 `ensemble abort main --hard --team main`（必要時）
- 驗收：至少有 `team`/`stream`/`control` 事件；至少 1 次 `LANDED` 或 `ESCALATED`。

### Slice C：五機配置
- `m1~m5` 都執行 `ensemble up`
- `m1` 上執行 `ensemble mesh` 與 `ensemble nodes`
- 主專案與 4 顆衛星各跑最小 run
- 驗收：`mesh` 可見本機 conductor 與 remote peers，`nodes` 可顯示 agent→host 輔助路由；任務可重跑成功。
- 對應工具：`scripts/phase2-fleet.ps1` 從未追蹤的 `phase2-fleet.local.json` 產生主專案與衛星專案 generated crew；先用 `-Materialize -PlanOnly` 預覽，再用 `-Materialize -RunSelected -Node <host>` 直接執行本節點選中的 run（`-RunSelected` 不接受預設 `all`），避免手工複製 crew、手工修改 `[agents.<name>] node` 路由，或手工轉貼 run 指令。

### Slice D：clean reinstall
- `pwsh scripts\uninstall.ps1 -RemoveMcpConfig -Repo . -Clients codex,claude,opencode`
- `pwsh scripts\install.ps1`
- `ensemble serve --install-service --print`
- `ensemble serve --uninstall-service --print`
- `ensemble up`
- `ensemble mesh`
- `ensemble nodes`
- `pwsh scripts\smoke.ps1 -NoBuild -Repo . -TimeoutSecs 120 -Reviewers claude -AllowEscalatedRun`
- `pwsh scripts\uninstall.ps1`
- 驗收：重裝後可再次進入網格，且 smoke 通過。

對應腳本：`pwsh scripts\phase2-verify.ps1 -Repo <repo> -SkipSliceB -SkipSliceC`

### English `/goal` form（direct paste）

```text
/goal complete Phase 2 cross-machine governance for a 5-node fleet while keeping single-machine regression green.

Acceptance:
- Single-machine: team status/say/inbox + team/watch/steer/abort + MCP team/control tools + controlled codex/claude/opencode/agy launch + supervise + install/uninstall + acceptance-single-machine.ps1 + smoke.ps1 must pass.
- Cross-machine: watch/steer/abort/team must support --node and member@node; --node local must force local; invalid node/token errors must fail explicitly and must not silently fall back to local.
- Governance invariant: cross-machine run still requires test pass + at least two distinct-vendor approvals (min_approvals=2) to land; conflicts must escalate, never auto-absorbed.
- Rebuildable service flow: uninstall -> install -> up/serve -> mesh -> nodes -> smoke can be repeated.
- Deployment scope: primary project on 5 nodes (m1 conductor) + 4 satellite projects (codex+claude), repeatable without manual file copy or manual routing.

Acceptance slices:
Slice A: ensemble team status --team main --json; ensemble nodes; ensemble watch codex@m1 --team main --since 0 --json
Slice B: ensemble run "<task>" --crew crew-main.toml --repo <repo> --team main --watch main; ensemble watch main --follow; ensemble steer/abort main as needed
Slice C: m1~m5 all run ensemble up, m1 runs mesh/nodes, primary + 4 satellites run minimal tasks and rerun cleanly; use scripts/phase2-fleet.ps1 with an untracked phase2-fleet.local.json to generate crew files and run the selected node's assigned task via -RunSelected instead of manually copying crew files, hand-editing agent node routes, or pasting generated run commands; mesh is the fleet visibility source of truth, while nodes is an agent→host helper view
Slice D: uninstall -> install -> serve --install-service --print -> serve --uninstall-service --print -> up -> mesh -> nodes -> smoke (pwsh scripts\smoke.ps1 -NoBuild -Repo . -TimeoutSecs 120 -Reviewers claude -AllowEscalatedRun) -> uninstall, with no residual errors
```
